# Gemini CLI 작업 지침 (Standard Operating Procedures)

이 파일의 지침은 프로젝트 내 모든 작업에서 최우선으로 적용됩니다.

## 1. 코드 품질 및 검증 (CRITICAL)
- **무조건 컴파일 검증**: 자바 코드를 수정하거나 새로운 파일을 생성한 경우, 답변을 마치기 전에 반드시 `./gradlew compileJava`를 실행하여 컴파일 에러가 없는지 확인해야 한다.
- **에러 자동 해결**: 컴파일 에러가 발생한 경우, 사용자에게 에러를 보고하기 전에 스스로 에러의 원인을 분석하고 코드를 수정하여 빌드가 성공하는 상태로 만든 뒤 최종 결과를 답변한다.
- **기능 검증**: 컴파일 성공 후, 구현하려던 기능이 의도대로 동작하는지 간단한 단위 테스트나 로직 검증을 수행한다.
- **환경 설정**: 빌드 시 반드시 Java 21 환경(`$env:JAVA_HOME = "C:\Users\lim\.jdks\corretto-21.0.3"`)을 명시적으로 설정하여 실행한다.

## 2. 형상 관리 및 자동화 (Git/GitHub)
- **단위별 커밋**: 적절한 기능 단위(Feature unit) 구현이 완료되고 빌드/테스트를 통과하면 즉시 Git 커밋을 수행한다.
- **자동 푸시**: 커밋 완료 후 원격 저장소(GitHub)에 즉시 `git push`를 수행하여 코드의 안전성을 확보한다.
- **커밋 메시지**: 구현한 기능을 명확히 설명하는 메시지를 작성한다. (예: `feat: add task conflict management logic`)

## 3. 기술 스택 및 스타일
- **Spring Boot 3.4.x / Java 21**
- **JSP (WEB-INF/views)** 기반의 UI
- **Lombok** 활용
- **JPA** 엔티티 연관 관계 및 데이터 무결성 유지

## 4. 답변 방식
- 작업 완료 후 "성공했습니다"라고 말하기 전에, 실제 빌드 로그와 Git Push 성공 여부를 다시 한번 확인한다.
- 모든 자동화 단계(빌드 -> 테스트 -> 푸시)가 완료된 후 최종 상태를 보고한다.
